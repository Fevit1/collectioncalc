### CSV Collection Import 🆕 HIGH PRIORITY
**Goal:** Remove switching barrier from competitor apps (CLZ Comics, ComicBase, League of Comic Geeks)

**Why This Matters:**
- Users with 10k+ comics in other apps won't manually re-enter
- Switching cost = massive barrier to trying SlabWorthy
- "Try us without losing your data" is powerful pitch
- Competitive advantage (most apps don't make this easy)

**Phase 1: Basic CSV Import**
- [ ] Upload CSV interface (drag/drop + file picker)
- [ ] Auto-detect common formats (CLZ Comics, ComicBase)
- [ ] Column mapping UI - Manual: "Their 'Title' → Our 'title'"
- [ ] Data preview (first 10 rows)
- [ ] Validation & error flagging
- [ ] Duplicate detection
- [ ] Bulk insert to `graded_comics` table
- [ ] Background job with progress bar (don't block UI for 10k rows)
- [ ] Import summary report (X imported, Y had issues)

**Phase 2: Enhanced Import**
- [ ] Platform-specific importers (auto-detect CLZ vs ComicBase format)
- [ ] Handle photos (if URLs provided in CSV)
- [ ] Smart field mapping (fuzzy match column names)
- [ ] Import history/audit log
- [ ] "Re-grade imported comics" feature (bulk upload photos)

**Supported Competitors:**
- CLZ Comics (CSV export) - Primary target
- ComicBase (CSV export)
- League of Comic Geeks (CSV download)
- Generic CSV (manual mapping)

**Data Mapping:**
```
CLZ Comics → SlabWorthy
Title → title
Issue # → issue_number
Publisher → publisher
Year → publication_year
Grade → grade (map text to numeric)
Purchase Price → (custom field, future)
Notes → notes
```

**Success Metrics:**
- X% of new users import existing collections
- Average import size (how many comics)
- Conversion rate: import → first grade → subscribe

**Goal:** Premium users can query market data using natural language

**Why This Matters:**
- Powerful research tool for serious collectors
- Differentiation from competitors
- Premium tier monetization opportunity
- Leverages our massive database (market_sales, ebay_sales, grading data)

**Example Queries:**
- "How many Absolute Batman #1 sales happened in the last week? What's the average price?"

- "What's the price trend for Amazing Spider-Man #300 over the last 6 months?"
- "Find all sales of Spawn #1 reprints vs first prints"

- [ ] Usage tracking for billing

**Pricing Considerations:**

- Enterprise: Unlimited queries
- Track API costs and adjust pricing accordingly

**Technical Notes:**

- Cache common queries to reduce API costs

### Analytics & Experimentation Platform

**Current State:** Basic admin console analytics only
**Goal:** Data-driven product decisions + automated optimization

#### Phase 1: PostHog Integration (Post-Beta)
**Why PostHog:** All-in-one (analytics + A/B tests + session replay), open source, generous free tier (1M events/mo)

- [ ] Add PostHog JS snippet to frontend
- [ ] Configure key events to track:
  - `grading_started` - User begins 4-photo flow
  - `photo_uploaded` - Each photo step (with step number)
  - `grading_completed` - Full flow completed
  - `grading_abandoned` - User exits mid-flow (with last step)
  - `verdict_shown` - "Worth the slab" or "Keep it raw" displayed
  - `save_to_collection` - User saves result
  - `signup_started` / `signup_completed`
- [ ] Set up conversion funnels:
  - Landing → Signup → First Grade → Save to Collection
  - Photo 1 → Photo 2 → Photo 3 → Photo 4 → Result
- [ ] Enable session replay for debugging UX issues
- [ ] Create dashboard for key metrics

**Key Metrics to Track:**
- Grading completion rate (% who finish all 4 photos)
- Drop-off by step (where do people abandon?)
- Time-to-grade (how long does full flow take?)
- Verdict distribution (what % are "worth the slab"?)
- Popular titles/issues being graded

#### Phase 2: Manual A/B Testing
- [ ] Set up first experiment: Tagline test
  - A: "Know before you slab"
  - B: "The pre-grade gut check"
  - C: "Should you slab it?"
  - Measure: Signup conversion rate
- [ ] Test CTA button copy ("Check a Comic" vs "Grade My Comic" vs "Get Started")
- [ ] Test pricing page layouts (when ready)
- [ ] Document experiment results and learnings

#### Phase 3: AI-Assisted Experimentation (Future)
**Vision:** AI proposes experiments, runs them, analyzes results, proposes iterations

**Architecture (integrate with RelevantSee platform):**
```
PostHog event data
    ↓
RelevantSee AI Engine
    ↓
Analyzes: What's underperforming? What patterns exist?
    ↓
Proposes: "Test headline X vs Y based on [reasoning]"
    ↓
Human approves (or auto-approve low-risk tests)
    ↓
Experiment runs via PostHog feature flags
    ↓
AI analyzes results, proposes next iteration
    ↓
Weekly digest to human: "Here's what we tested and learned"
```

**Candidate areas for AI-driven testing:**
- Headlines and taglines
- CTA button text and colors
- Email subject lines
- Onboarding flow sequence
- Feature flag rollouts

**Shared Service Vision:**
This becomes a RelevantSee microservice that Slab Worthy, PSAWorthy, CoinWorthy, etc. all call. One AI experimentation engine powering the whole "Worthy" brand family.
