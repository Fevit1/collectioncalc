## ARCHITECTURE OVERVIEW

### Backend (Flask/Python)
- **wsgi.py** - Main app (~1,900 lines, will refactor at 2,500)
- **Blueprints:** auth_routes.py, admin_routes.py, sales.py
- **Database:** PostgreSQL on Render
- **APIs:** Claude API (grading), Cloudflare R2 (images), Resend (email)

### Frontend
- **HTML/CSS/JS** - Custom responsive design
- **No frameworks** - Vanilla JS for simplicity
- **Mobile-first** - Works on all devices

### Data Pipeline
1. User uploads 4 photos
2. Cloudflare R2 stores images
3. Claude Vision API extracts title/issue/year
4. Claude analyzes defects (spine ticks, color breaks, etc.)
5. Custom grading algorithm assigns CGC-equivalent grade
6. PostgreSQL stores result
7. FMV calculated from sales database (eBay + Whatnot + market data)

### Chrome Extensions (Separate Repos)
- **eBay Collector** - Scrapes eBay sold listings
- **Whatnot Collector** - Scrapes Whatnot auction data
- **Extension shared:** Uploads data to backend API

## DATABASE STRUCTURE

### Key Tables
- `users` - Authentication, email verification
- `graded_comics` - User's graded comics with photos
- `market_sales` - Historical sales data (generic)
- `ebay_sales` - eBay sold listings with normalization
- `ebay_sales_staging` - Raw eBay data before normalization
- `whatnot_sales` - Whatnot auction results
- `comic_registry` - 🆕 Coming soon (fingerprint registry)

### Normalization Fields (ebay_sales)
- `canonical_title` - Standardized series name
- `issue_number` - Parsed issue number
- `variant_info` - Cover variant details
- `creator_info` - Writer/artist attribution
- `publisher` - Marvel, DC, Image, etc.
- `condition_grade` - Extracted grade
- (13 total normalized fields)
