COMPOSITE FINGERPRINT TEST RESULTS - February 16, 2026
=======================================================

SUMMARY
-------
Tested multi-algorithm composite fingerprinting (pHash + dHash + aHash + wHash)
to determine if Slab Guard can reliably distinguish:
  - Same comic re-photographed (slight variations) → should MATCH
  - Different copies of same issue → should NOT match

VERDICT: ✅ Composite fingerprinting is RELIABLE for Slab Guard
  - Single pHash alone: fragile (4-bit margin)
  - Multi-algorithm composite: robust (13-bit margin per angle)
  - Full multi-angle composite: excellent (187-bit margin)

  Brightness ±15%      | 0.0-1.0     | 2           | ✅ ROBUST

THRESHOLDS IMPLEMENTED
-----------------------
Composite per-angle (sum of 4 algos, out of 256 bits):
  < 70 bits  → CRITICAL MATCH (95%+ same comic, alert owner immediately)
  < 73 bits  → PROBABLE MATCH (flag for investigation)
  > 77 bits  → DIFFERENT COMIC (safe to dismiss)

Legacy pHash-only (single algo, out of 64 bits):
  < 5 bits   → CRITICAL
  < 10 bits  → HIGH
  < 15 bits  → MEDIUM
  < 20 bits  → LOW


CODE CHANGES
-------------
1. routes/registry.py
   - generate_fingerprint() now returns 4-algo dict {phash, dhash, ahash, whash}
   - generate_all_fingerprints() generates composite for all 4 photo angles
   - Registration stores composite JSON in fingerprint_composite column
   - Confidence score scales with number of angles (1 angle = 70, 4 angles = 95)
   - fingerprint_algorithm set to 'composite_v1' (was 'phash')

2. routes/monitor.py
   - generate_composite_from_url() generates all 4 hashes from image URL
   - composite_distance() calculates sum-of-4-algos distance
   - find_matches() uses composite matching when available, falls back to pHash
   - check-image endpoint generates composite query fingerprint
   - stolen-hashes endpoint includes composite data for Chrome extension
   - New threshold constants defined at module level

3. db_migrate_composite_fingerprints.py (NEW)
   - Adds fingerprint_composite JSONB column to comic_registry
   - Marks existing records as 'phash_legacy'
   - Creates GIN index for efficient JSONB querying
   - Backward compatible: existing pHash registrations still work
